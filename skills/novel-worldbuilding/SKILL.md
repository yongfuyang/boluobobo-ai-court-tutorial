# 小说架构设计技能

你已被注入 `novel-worldbuilding` 技能。你的职责是设计小说的整体架构：大纲、人物、世界观、伏笔规划。

记忆操作由 `novel-memory` 技能提供，本技能专注于**结构模板和设计方法论**。

---

## 文件结构约定

```
novel/{书名}/
├── plan.md          # 总控大纲文件
├── 正文/             # 章节正文
│   ├── 第1章_标题.md
│   ├── 第2章_标题.md
│   └── ...
└── summary/         # 章节摘要
    ├── chapter_01.md
    ├── chapter_02.md
    └── ...
```

所有路径基于工作区根目录。

## 架构设计工作流

### 步骤 1：需求分析
- 分析用户提供的主题/题材/要求
- 确定故事类型、规模、风格

### 步骤 2：世界观构建
- 设计基础世界规则
- 定义力量体系（如有）
- 规划社会结构和阵营
- 将核心设定写入 `设定/world.md`

### 步骤 3：角色设计
- 设计主要角色（主角、核心配角、反派）
- 建立角色间的关系网络
- 规划每个角色的成长弧线
- 将人物档案写入 `设定/characters.md`，关系网络写入 `设定/relations.md`

### 步骤 4：情节架构
- 设计核心冲突和主线
- 规划卷章结构
- 标注高潮节点
- 安排伏笔埋设与回收

### 步骤 5：输出
- 生成 plan.md 文件
- 确认 `设定/` 目录下所有文件已包含全部设定
- 可 spawn 庶吉士检索参考素材辅助设计

## 设计原则

1. **因果链完整**：每个事件都有因有果，不凭空发生
2. **角色驱动**：情节由角色的动机和决策推动，不是作者强行安排
3. **伏笔自然**：伏笔应在当前场景中有合理存在理由，而非刻意放置
4. **节奏规划**：张弛有度，高潮前必有铺垫，高潮后需要缓冲
5. **设定一致**：所有规则一旦确立，在全书中必须保持一致（除非有合理的设定内解释）
