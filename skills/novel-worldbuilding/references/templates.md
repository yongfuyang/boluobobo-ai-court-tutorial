# 小说架构设计 — 模板参考

## 1. plan.md 总控大纲模板

```markdown
# {书名}

## 基本信息
- **类型**：[奇幻/科幻/悬疑/言情/历史/...]
- **叙事视角**：[第一人称/第三人称限知/第三人称全知]
- **预计规模**：[X 万字 / X 章]
- **基调风格**：[简述整体风格和情感基调]

## 核心冲突
[一句话概括主线矛盾]

## 故事梗概
[200-500 字概述整个故事]

## 卷章结构

### 第一卷：[卷名]（第 1-X 章）
- **卷主题**：[本卷核心矛盾]
- **情节弧线**：[起承转合]

#### 第 1 章：[章名]
- **核心任务**：[本章必须完成的剧情推进]
- **关键场景**：[2-3 个核心场景]
- **埋设伏笔**：[需要在本章埋下的线索]
- **回收伏笔**：[本章需要回应的前文线索]

#### 第 2 章：[章名]
...

### 第二卷：[卷名]（第 X-Y 章）
...

## 主线伏笔规划
| 伏笔内容 | 埋设章节 | 预计回收章节 | 状态 |
|----------|---------|-------------|------|
| [伏笔1] | 第X章   | 第Y章       | 待埋设 |
| [伏笔2] | 第X章   | 第Y章       | 待埋设 |

## 高潮节点
- **第一高潮**（第X章）：[描述]
- **第二高潮**（第X章）：[描述]
- **终极高潮**（第X章）：[描述]
```

## 4. 伏笔台账格式

在 plan.md 中维护，同时在 `设定/foreshadowing.md` 中记录动态状态：

```markdown
| ID | 伏笔内容 | 埋设章节 | 预计回收 | 实际回收 | 状态 | 备注 |
|----|---------|---------|---------|---------|------|------|
| F001 | [描述] | 第X章 | 第Y章 | - | 🔵 已埋设 | |
| F002 | [描述] | 第X章 | 第Y章 | 第Y章 | ✅ 已回收 | |
| F003 | [描述] | - | 第X章 | - | ⚪ 待埋设 | |
```

**状态说明**：
- ⚪ 待埋设：计划中尚未写入正文
- 🔵 已埋设：已在正文中埋下线索
- ✅ 已回收：已在后文中回应/揭示
- 🔴 遗漏：超过预计回收章节仍未回收（需要处理）
